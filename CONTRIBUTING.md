# 贡献指南

感谢您有兴趣为 TrBlog 项目做出贡献！

## 如何贡献

### 报告问题
- 请在 GitHub Issues 中报告问题
- 尽可能详细地描述问题
- 提供复现步骤
- 描述您的环境

### 提交代码
1. Fork 本仓库
2. 创建一个新分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的更改 (`git commit -m 'feat: Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 发起一个 Pull Request

## 代码风格
- 前端：使用 Prettier 和 ESLint
- 后端：遵循各自的代码风格规范
- 提交信息使用 Conventional Commits 规范

## 提交信息格式
```
<type>(<scope>): <subject>
```

### 类型（type）：
- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 格式调整
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 其他变更

## 开发流程
1. 确保您已经阅读了 [DEVELOPMENT.md](DEVELOPMENT.md)
2. 确保所有测试通过
3. 确保代码符合项目风格规范
4. 更新文档（如有必要）

## 行为准则
我们致力于提供一个友好和安全的环境，请尊重其他贡献者。

## 联系方式
如有任何问题，请联系我们。
