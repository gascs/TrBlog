# 部署检查清单

## 1. 环境变量配置

### 服务器端 (.env)
- [ ] 更改 `JWT_SECRET` 为强随机值
- [ ] 将 `NODE_ENV` 设置为 `production`
- [ ] 配置正确的 `DATABASE_URL`
- [ ] 配置正确的 `REDIS_URL`
- [ ] 设置正确的 `FRONTEND_URL`
- [ ] 配置 `PORT` 为合适的端口

### 客户端 (.env)
- [ ] 设置 `VITE_API_URL` 为生产环境的 API 地址
- [ ] 设置 `NODE_ENV` 为 `production`
- [ ] 确保 `REACT_APP_MOCK_MODE` 为 `false`

- [ ] 检查 origin check 中间件配置

## 8. 部署步骤

1. **准备环境**
   - 确保服务器满足系统要求
   - 安装必要的依赖

2. **配置环境变量**
   - 按照上述检查清单配置环境变量

3. **构建应用**
   - 服务器端：`npm run build`
   - 客户端：`npm run build`

4. **部署应用**
   - 使用 Docker Compose 部署
   - 或使用其他部署方式

5. **验证部署**
   - 检查健康检查端点
   - 测试主要功能
   - 验证性能和安全性

6. **监控部署**
   - 监控系统运行状态
   - 检查日志
   - 确保系统稳定运行
